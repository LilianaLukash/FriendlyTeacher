# Friendly Sam — запуск

## 1. Установка зависимостей

**Приложение (Expo):**
```bash
npm install
```

**Бэкенд (API):**
```bash
cd server
npm install
cp .env.example .env
```
В `.env` укажи `OPENAI_API_KEY=sk-...` (ключ OpenAI).

## 2. Запуск

**Терминал 1 — бэкенд:**
```bash
cd server
npm run dev
```
API будет на http://localhost:3000

**Терминал 2 — приложение:**
```bash
npm start
```
Отсканируй QR-код приложением **Expo Go** на телефоне (телефон и компьютер в одной Wi‑Fi сети).

**Продакшен API (Railway):** в корне проекта в `.env` уже можно указать:
```
EXPO_PUBLIC_API_URL=https://friendlyteacher-production.up.railway.app
```
**Если тестируешь локальный бэкенд на телефоне (Expo Go):** укажи IP компьютера:
```
EXPO_PUBLIC_API_URL=http://192.168.1.XXX:3000
```
(Узнать IP: в PowerShell `ipconfig`, смотри IPv4.)

### Railway (порт — только на стороне сервера)

Приложение и `EXPO_PUBLIC_API_URL` используют **только** `https://…up.railway.app` **без порта** — менять порт в Railway **не ломает** клиент.

- В **`server/.env.example`** указан `PORT=3000` — это **только для локальной** разработки на своём ПК.
- В проде сервер слушает **`process.env.PORT`**, который задаёт **Railway** (в Deploy Logs будет строка вида `Friendly Sam API on port …`).
- Чтобы сайт открывался: в Railway → **Settings → Networking** у публичного домена **целевой порт должен совпадать** с этим числом из логов (если в логах `8080`, в UI тоже `8080`). Переменную **`PORT` в Railway Variables лучше не задавать вручную**, пока всё работает — тогда платформа сама выставит нужное значение.

## 3. Что уже есть

- **Онбординг** — имя, выбор языка (English / Português PT). Показывается один раз при первом запуске.
- **Экран Home** — **Старт** и **Мои фразы**.
- **Сессия** — поле ввода + «Отправить». Текст уходит в API (Sam/LLM), ответ приходит и озвучивается (TTS). Совпадения с фразами из БД (learning/review) считаются повторениями. Показаны токены за сессию.
- **Мои фразы** — список фраз с переводом, «Послушать» и «Повторить» (TTS).
- **Бэкенд** — `server/`: POST `/chat` (transcript + memoryContext + targetLang) → LLM Sam → reply + tokensUsed. Подсчёт токенов с первого запроса.
